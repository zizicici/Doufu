# 执行计划

## Phase 0：文档与方向对齐（已开始）

1. 明确愿景、范围、主页面规格。
2. 固化工程约束与模块边界。
3. 产出可交接文档入口。

## Phase 1：主页面骨架（当前）

1. 实现 Home 页面布局与两个核心按钮。
2. 实现项目列表空状态。
3. 预留导航动作（新建 / 打开）。

验收标准：
1. 启动后首页可正常显示。
2. 两个按钮可点击并触发动作。

## Phase 2：本地项目管理

1. 定义 `AppProject` 模型与存储目录规范。
2. 实现项目创建与列表读取。
3. 支持删除项目（带确认）。

验收标准：
1. 创建后列表可见并可再次打开。
2. 重启 App 后项目仍存在。

## Phase 3：网页运行与编辑入口

1. 使用 `WKWebView` 加载项目入口页。
2. 提供基础刷新和错误提示。

验收标准：
1. `index.html` 可在 App 内正确渲染。
2. 文件缺失或加载失败时有明确提示。

## Phase 4：LLM 代码生成接入

1. 封装生成服务接口与重试策略。
2. 生成 `html/css/js` 并落盘。
3. 记录生成版本与时间戳。

验收标准：
1. 输入需求后可生成可运行页面。
2. 失败场景具备可恢复路径。

## Phase 5：分享与安全强化

1. 设计可控导出流程。
2. 强化安全策略与内容校验。

验收标准：
1. 导出不破坏本地沙盒隔离。
2. 分享流程明确提示隐私边界。
