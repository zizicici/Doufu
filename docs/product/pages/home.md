# 页面规格：主页面（Home）

## 页面目标

作为用户进入 App 后的第一个页面，用“项目画廊”方式直接管理和进入本地项目。

## 页面结构

1. 顶部导航栏
   - 标题：`豆腐`
   - 左侧：`设置`按钮（齿轮）
   - 右侧：`新建`按钮（`+`）
2. 搜索栏
   - 占位文案：`搜索项目`
   - 仅在存在项目时显示；无项目时隐藏。
3. 项目网格
   - 卡片平铺展示（约每行 3 列）。
   - 卡片内容：项目预览图（若有）、项目名、更新时间。
4. 空状态层
   - 无项目：`还没有网页项目，点击右上角 + 创建第一个吧。`
   - 搜索无结果：`没有匹配的项目，换个关键词试试。`

## 交互说明

1. 点击右上角 `+`
   - 立即创建本地空白项目。
   - 自动进入项目运行页。
2. 点击项目卡片
   - 进入该项目运行页。
3. 长按项目卡片
   - 弹出上下文菜单：`设置`、`排序`、`删除`。
4. 点击 `排序`
   - 进入独立排序页，支持拖拽调整项目顺序。
5. 点击 `删除`
   - 二次确认后删除项目目录并刷新首页列表。
6. 输入搜索关键词
   - 对项目名和描述进行实时过滤。

## 布局建议（UIKit）

1. 使用 `UICollectionViewFlowLayout`，网格项宽度按屏幕约 1/3 计算。
2. 卡片为无描边、圆角、轻层级视觉。
3. 首页背景使用系统背景色，避免额外装饰干扰项目内容。
4. 空状态覆盖层约束在 `safeAreaLayoutGuide` 内。

## 异常与边界状态

1. 无项目：隐藏搜索栏，仅显示空状态文案。
2. 项目读取失败：弹出错误提示，并保持当前页面可继续操作。
3. 项目数量较多：支持滚动；导航栏按钮始终可达。
