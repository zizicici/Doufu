# 产品愿景

## 一句话愿景

让普通用户无需传统编程背景，也能通过自然语言在 iPhone 上本地创建、运行并持续迭代小型 App（基于 `html + css + js`）。

## 核心目标

1. 降低创建门槛：用户描述需求后，能快速得到可运行页面并继续通过聊天修改。
2. 移动端优先：默认以 iPhone 竖屏和 Safe Area 作为首要设计约束。
3. 本地可控：项目默认在本地创建、运行、编辑，降低外部依赖。
4. 安全优先：敏感凭证使用系统安全存储，项目数据在沙盒内隔离。

## 目标用户

1. 想把想法快速做成可用工具、但不熟悉传统开发流程的普通用户。
2. 在手机上高频试错、迭代界面的个人创作者。
3. 需要低成本原型验证、并可后续导出或分享的轻量开发者。

## 产品原则

1. `Local First`：核心体验在离线/本地也应可用。
2. `Prompt to Product`：从想法到可运行页面路径尽可能短。
3. `Safe by Default`：默认最小权限、默认隔离执行。
4. `Mobile Native Feel`：默认视觉与交互尽量贴近 iOS 原生观感，避免强网页感。
5. `Gradual Power`：新手可用，进阶用户可逐步控制细节。

## 北极星指标（建议）

1. 新用户首次创建成功率（含首次进入项目运行页）。
2. 从输入需求到首个可运行页面的平均时长。
3. 首次聊天改动成功率（模型返回可应用变更）。
4. 7 日内复访并继续编辑项目的比例。
5. 移动端可用性指标（布局越界率、触控可达性问题率）。
